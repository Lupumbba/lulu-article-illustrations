# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有噜噜。
- 噜噜是暖黄橙水豚气质，头顶有橘子或清晰橙色识别点。
- 噜噜承担核心动作，不只是装饰。
- 画面温吞、稳定、轻怪诞、有创意。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于噜噜识别点或主路径/箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。
- 没有直接复刻旧案例构图，而是为当前文章生成了新隐喻。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 噜噜只是站在角落、举牌或卖萌。
- 噜噜像真实水豚照片、复杂动漫角色、表情包或儿童卡通。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有物理隐喻。
- 暖黄橙色铺满全图，导致角色识别点不突出。

## 迭代方法

- 太普通：让噜噜成为动作主体，加入一个温吞但成立的物理隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 calm blank expression、emotionally stable、not a sticker mascot、not childish。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像表情包：缩小大头，减少夸张表情和装饰，增加文章结构动作。
- 太不像噜噜：强化暖黄橙水豚身体、豆豆眼、头顶橘子、短小四肢。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“很稳、但有点奇怪”，然后 1 秒内看懂结构。

如果第一眼像教程页、表情包或商业插画，就不合格。
