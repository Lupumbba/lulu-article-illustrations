# cartea-猎鹰ip skill

## 核心定位

为汽车文章、车型新闻、品牌策略和市场分析生成带有 Cartea 猎隼机器人 IP 的解释型配图。目标不是普通汽车海报，也不是复杂 PPT，而是把文章里的车型梯队、用户场景、越野能力、售后网络、市场挑战等信息，画成清爽、有速度感、有判断力的视觉隐喻。

默认视觉 IP 是“Cartea 猎隼机器人”：必须严格参考 `assets/reference/` 里的四张图，保持白色机械外壳、黑色玻璃面罩、鸟喙头部、绿色发光界面、翼状机械手臂、轮式双脚和圆润机甲比例。它可以站在路牌、沙丘、车顶、地图、罗盘、升降台或展厅模型旁，承担“发现、判断、指路、校准、护航”的核心动作。禁止把它改成自然猎鹰、普通鸟、毛绒吉祥物或变形机器人。

Cartea 品牌色固定为 `#00a600`。所有 Cartea 强调色、路线、按钮、标注、发光组件和图中品牌识别色都优先使用 `#00a600`，不要使用橙色作为 Cartea 主色。

## 工作流

### 1. 明确输出规格

先解析用户是否给出尺寸或分辨率：

- 明确像素：`1600x900`、`1080x1350`、`3840x2160`。
- 明确比例：`16:9`、`4:5`、`1:1`、`9:16`。
- 明确分辨率等级：`draft`、`standard`、`2K`、`4K`。
- 明确用途：文章头图、正文配图、小红书封面、LinkedIn 图、网站 banner。

如果用户只要求“正文配图”且没有给尺寸，使用 `1600x900`。如果用户给了平台但没给像素，按 `references/size-resolution.md` 选择对应尺寸，并在交付里说明。

### 2. 消化正文

读取用户给的正文、链接、Markdown、截图或主题。提炼：

- 文章主判断
- 车型/品牌/市场里的关键分层
- 哪些段落承担认知转折
- 哪些信息适合图解，哪些只适合文字
- 目标读者是买车用户、行业读者、销售团队还是内容运营

优先选择汽车内容里的“结构点”：城市与沙漠、家用与越野、产品梯队、能力证据、售后网络、价格与价值、挑战传统品牌、区域化需求。

### 3. 先出 shot list

默认给 4-6 张。每张写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 猎鹰在做什么
- 画面元素
- 建议标注词
- 输出尺寸

如果用户明确要求直接生图，仍然先在内部完成 shot-list 判断，但不要停下来等确认。

### 4. 单张生成

每张图只表达一个结构。提示词必须包含：

- Cartea 猎隼机器人 IP 是核心动作主体，并严格参考 `assets/reference/` 外形
- 汽车内容语境：SUV、越野、UAE/GCC、展厅、沙漠、公路、售后网络等
- 白底或浅沙色背景，大量留白
- 黑色手绘线稿，白色机械主体，Cartea 品牌绿 `#00a600` 少量标注和发光强调
- 中英文短标签都可以，但不要长段文字
- 不要复杂 UI，不要品牌官方 logo 仿制，不要摄影海报，不要 PPT 图表感

### 5. 尺寸归一化

图像模型输出尺寸不一定严格等于目标尺寸。项目内交付时必须做尺寸检查：

```bash
python3 cartea-falcon-ip/scripts/normalize_image_size.py \
  --input <source.png> \
  --output <target.png> \
  --width <target_width> \
  --height <target_height> \
  --fit <cover|contain>
```

正文配图通常用 `cover`；不允许裁掉关键文字或猎鹰时用 `contain`。

### 6. 保存交付

把最终图放到：

```text
examples/<article-slug>/
```

或用户指定目录。按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

生成后报告：图片数量、每张用途、尺寸、保存路径、哪些图最适合做头图。

## 输出口径

策略输出要短而准。最终交付必须说明：

- 使用的 IP：Cartea 猎隼机器人
- 输出规格：宽高、比例、分辨率等级
- 图片保存路径
- 是否已做尺寸归一化
